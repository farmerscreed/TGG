This document records a full audit of the participant, judging, admin,
coordinator, and team flows, cross-referencing every database call in the

## 1. Database schema reference

Public tables and their key columns (as of this audit):

| Table | Primary key | Auth link | Notable columns |
|---|---|---|---|
| `user_roles` | `id` | `id` = `auth.users.id` | `role`, `university`, `is_active` |
| `profiles` | `id` (own uuid) | **`user_id`** → `auth.users.id` | `first_name`, `last_name`, `phone`, `gender`, `university`, `department`, `year_of_study`, `participation_type`, `profile_photo_url` |
| `submissions` | `id` | `user_id` → `auth.users.id` | `reference_code`, `team_id`, `title`, `category`, `status`, `is_locked`, `current_step` |
| `submission_files` | `id` | via `submission_id` | `file_url` (storage path), `file_type`, `file_name`, `file_size_bytes` |
| `teams` | `id` | `lead_user_id` → `auth.users.id` | `name`, `university` |
| `team_members` | `id` | `user_id` → `auth.users.id` | `team_id`, `invited_email`, `status`, `invite_token` (auto-generated) |
| `judge_assignments` | `id` | `judge_id` | `submission_id` |
| `judge_scores` | `id` | `judge_id` | `submission_id`, `criterion_id`, `score`, `comments` — **one row per criterion**; unique `(judge_id, submission_id, criterion_id)` |
| `judging_criteria` | `id` | — | `name`, `description`, `max_points`, `weight`, `display_order`, `is_active` |
| `notifications` | `id` | `user_id` | `title`, `message`, `is_read` |
| `challenge_settings` | `id` | — | `registration_*`, `submission_open`, `submission_deadline`, `judging_*`, `results_date`, `judging_locked` (single row) |
| `challenge_categories` | `id` | — | `name`, `description`, `is_custom`, `display_order`, `is_active` |
| `submission_notes` | `id` | `author_id` | `submission_id`, `note` |

### Key relationship gotchas

- **`profiles` is keyed by `user_id`, not `id`.** Any `profiles.eq('id', authUserId)`
  is wrong — use `.eq('user_id', authUserId)` and `onConflict: 'user_id'` for upserts.
- **`user_roles` *is* keyed by the auth id** (`user_roles.id = auth.users.id`).
- **There is no direct FK between `user_roles` and `profiles`** (both point at
  `auth.users`). You cannot embed one into the other via PostgREST — fetch both
  and merge by id in code.
- To embed `profiles` from `submissions`, use the FK
  **`submissions_profile_user_fkey`** (`submissions.user_id → profiles.user_id`).
  The similarly-named `submissions_user_id_fkey` points to `auth.users` and does
  **not** work for embedding profiles.
- Storage bucket `submission-files` is **private** → serve files with signed URLs.

### Verified correct (no change needed)

- `profiles` upsert in `participant/profile/profile-form-client.tsx` (uses `user_id` + `onConflict`).
- `admin/submissions/page.tsx` and `admin/submissions/[id]/page.tsx` profile embeds (correct FK).
- Judge dashboard assignment embed (`judge_assignments → submissions`).
- Team `invite_token` auto-generates via DB default; `reference_code` via `set_reference_code` trigger.
- `challenge_settings` save (all form fields are valid columns).
- RPCs `get_admin_participants` and `get_participant_detail`.
- `user_roles` access (keyed by `id`) throughout.

### Open — not applied (recommendations)

- **`submissions.user_id` has no unique constraint.** Autosave races could create
  duplicate drafts, which then break the one-row assumption on load. Consider a
  unique index (individual entries) or a partial constraint if team entries are added.
- **No `custom_category` validation** on submit when category = "Other".
- **Build hard-depends on `RESEND_API_KEY`** at module load (`lib/email/index.ts`);
  a missing key fails the entire build, not just email. Vercel has the key, so
  production is fine, but consider lazy-initialising the Resend client.
- **Security advisors** (all WARN): enable leaked-password protection; restrict
  `anon`/`authenticated` EXECUTE on `get_admin_participants` / `get_participant_detail`;
  set `search_path` on flagged functions; tighten `profile-photos` public listing.
